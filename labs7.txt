1. Запусти контейнер с названием alpine-100mb из образа alpine, ограничив его память 100mb.
Не забудь, что alpine это ОС, внутри должна работать какая-то команда.
Проверить себя ты можешь с помощью команды docker stats --no-stream

docker run --name alpine-100mb --memory=100m -d alpine sh -c "while true; do echo 'Running...'; sleep 5; done"

2. Запусти контейнер в detached mode с именем http из образа nginx:alpine
Иногда для целей отладки мы можем предоставить контейнеру доступ в namespaces докер-хоста. Это считается плохой практикой, поскольку мы разрушаем всю модель безопасности Docker. Вместо этого, мы можем предоставить инструментам профилирования доступ в namespaces, связанных с контейнером.

docker run --name http -d nginx:alpine

3. Запусти команду:
docker run --net=container:http benhall/curl curl -s localhost
Контейнеры-отладчики теперь могут использовать пространство имен контейнера http. Они окажутся как бы в одной песочнице, и новый контейнер сможет обратиться к веб-серверу просто через localhost.
